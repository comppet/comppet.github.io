Um dado Gigante

[bisaopaulo]

Big Data é um Dado muito grande?

A resposta para essa pergunta é sim e não. Primeiramente não porque o termo Big Data não pode ser usado para tratar apenas um dado que tenha um tamanho enorme. Em segundo lugar a resposta é sim pois Big Data é definido como um conjunto de dados, não apenas um dado, que podem possuir ou não um grande tamanho, mas que em conjunto formam uma imensidão de conhecimento armazenados, que precisam de ferramentas específicas para serem manipulados, extraindo desses dados qualquer informação útil à quem as usa.

À grosso modo, mas de uma maneira mais simples, o conceito de Big Data pode ser entendido como uma análise de uma enorme quantidade de dados, para a geração de mais informações que também são importantes, só que em um volume menor, sendo mais amigável ao entendimento humano.

Não é estranho pensar que essa abordagem, a de se ter ferramentas para manipular inimagináveis quantidades de dados seja necessária, afinal de contas, quantos emails trocamos por dia? Quantas mensagens SMS ou em outros aplicativos você troca com seus conhecidos? Quantas bilhões de transações bancárias são feitas diariamente? Estes dados não simplesmente surgem de algum lugar e são descartados logo que são usados ou chegam ao seu destino. E com essa gigantesca quantidade de dados sendo armazenadas ou repassadas, temos que ter uma ferramenta capaz de manipulá-los e nos ajudar no meio dessa imensidão caótica de informação.

Os 5 V's do Big Data


Um dos vários Data Centers da Microsoft

[ArchDaily]

Para esclarecer melhor a ideia do conceito de Big Data, alguns especialistas passaram a tratar o tema em características que descrevem melhor o uso de Big Data, estas características são os cinco V's: -volume, velocidade, variedade, veracidade e valor.

  • Volume: como já citamos antes, o volume de dados quando falamos em Big Data é realmente grande. Esses dados crescem exponencialmente e justamente por isso são reutilizados;

  • Velocidade: é outro ponto que você já deve ter percebido a importância. Para evitar muitos problemas e maximizar a eficiência do Big Data, a sua manipulação, isto é (obtenção, gravação, atualização, ...), deve ser feita com um tempo muito curto, muitas vezes em tempo real para garantir a consistência da informação. Imagine quantos problemas uma operadora de cartões de crédito não teria se não conseguisse aprovar ou negar transações em tempo real;

  • Variedade: é um outro aspecto de grande importância. As enormes quantidades de dados que são geradas hoje, são também uma consequência da diversidade de informações. Uma variedade infinita de tipos de dados diferentes, tanto os que podem ser armazenados em bancos quanto os que não são estruturados e têm inúmeras fontes na rede, como áudios, vídeos e diversos documentos;

  • Veracidade: a confiabilidade dos dados é um aspecto também fundamental para o conceito do Big Data. Não adianta ter um grande volume de dados, uma boa velocidade de tratamento, e uma grande variedade de dados se os dados são falsos ou não confiáveis. Por isso também devemos considerar manter a consistência dos dados para que a informação extraída seja proveitosa;

  • Valor: a informação é um patrimônio. Como veremos nas próximas seções desta edição, o que traz a maioria dos interesses voltados para o Big Data, é o seu alto potencial como arrecadador de mercado para uma empresa que se utiliza das informações geradas para melhorar seu negócio;
