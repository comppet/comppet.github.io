Usando o VirtualBox(){
Software Livre

Por: Vinícius Fonseca Maciel


Se você está em um curso que abrange alguma área da computação possivelmente já ouviu professores ou amigos comentarem sobre esse software. Certamente você deverá ter ouvido algo como, “Com esse programa é possível ter Ubuntu e Windows em uma mesma máquina”, bom isso serve como uma espécie de introdução do que você irá ler logo abaixo. Se nunca ouviu, é uma boa oportunidade de entender sobre essa ferramenta fantástica que você pode ter acesso facilmente na internet.

VirtualBox pela Oracle

Inicialmente desenvolvido pela Innotek e Sun Microsystems, posteriormente adquirido pela Oracle, hoje o VirtualBox é um dos softwares mais utilizados no ramo de "simuladores" de sistemas operacionais. Conhecido, principalmente, pela sua portabilidade, licença de distribuição e acessibilidade, este aplicativo promete e cumpre o seu papel de auxiliar pessoas a ampliarem seus conhecimentos sobre informática sem comprometer os arquivos dos seus computadores reais. Essa característica permite que o usuário, sem gasto de tempo, instale, desinstale, modifique configurações de sistema e altere o funcionamento de programas sem ter o compromisso de que este deverá ser o seu ambiente permanente de trabalho.


  • Mas o que realmente é o VirtualBox?

O VirtualBox é um programa implementado usando conceitos de máquina virtual, ou seja, ele visa criar uma espécie de computador que não utiliza diretamente recursos físicos, mas sim recursos abstratos. É como se você tivesse dentro do seu computador um outro que funcionasse ao mesmo tempo de forma semelhante, no qual você pudesse salvar arquivos independentes (virtualizados), executasse programas e tivesse múltiplos sistemas operacionais. Por exemplo, você poderia desenvolver uma tarefa no Windows ao mesmo tempo em que rodaria uma versão de Ubuntu em outro ambiente no mesmo computador, que você acessaria com apenas alguns cliques.

Na imagem logo abaixo você pode ver um exemplo em que um computador real, que opera com MAC OS X, simula uma versão do Windows 7 em uma janela.

Virtualbox em MAC OS X

As áreas de aplicação deste software vão além do que apenas usuários comuns podem usufruir. Esta ferramenta visa atingir, inclusive, o meio acadêmico e instituições de ensino para informática e manutenção. Por poder ser instalado tanto em Linux, quanto Windows, Solaris ou Mac OS X isso facilita para que instituições, independentemente do sistema operacional disponível, façam demonstrações de outras plataformas, ou aplicativos destas, e aplicar o VirtualBox como via para o aprendizado.

  • Como são definidas as licenças para o uso do VirtualBox?

Nos anos de 2007 e 2008 quando a Innotek fora incorporada pela Sun Microsystems, o VirtualBox se tornou um programa quase que totalmente livre, ou melhor dizendo, de dupla-licença. Uma é a conhecida versão OSE, ou Open Source Edition, a qual possui código aberto, é gratuito e segue os termos da licença GPL(General Public License) a outra é a versão PUEL, ou Personal Use and Evaluation License, que designa uma licença para uso pessoal e avaliação por um período de tempo. As diferenças entre eles estão basicamente na forma como são distribuídos, o OSE possui apenas as funções principais e o PUEL possui algumas “implementações extras”, que não possuem seus códigos revelados.

Quando a empresa Sun Microsystems se tornou subsidiária da Oracle, algumas modificações foram realizadas em relação à OSE e à PUEL. Antes eram disponibilizados dois programas diferentes, um para cada tipo de licença, hoje o usuário faz o download do aplicativo do tipo OSE e, posteriormente, decide se deseja alguns acréscimos que só podem ser adquiridos caso se obtenha o pacote de expansão por outro link, este último permanece com a licença PUEL.

  • É difícil instalar o VirtualBox?

A instalação é bem intuitiva e a interface do programa amigável, basta o usuário escolher um Sistema Operacional e seguir os passos ajustando suas preferências para a virtualização. Na instalação é possível, inclusive, configurar uma memória e um disco rígido virtuais, de acordo com as preferências. É claro que quanto mais elas forem exigentes, mais será exigido do hardware. Daí em diante poderá ser feita a instalação da plataforma do mesmo modo que em um computador real.
